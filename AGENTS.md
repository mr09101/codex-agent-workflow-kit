# Agent Operating Guide

This file describes how AI coding agents should work in this repository. It is public-safe and applies to the repository itself. The template copied into generated projects lives at `templates/AGENTS.md`.

## Purpose

Codex Agent Workflow Kit is a dependency-free Python CLI and documentation starter kit for maintainers using Codex or other AI coding agents. Agents working here should keep the project small, auditable, and easy to maintain.

## Safety Rules

- Do not add secrets, tokens, credentials, `.env` files, private paths, account details, or production data.
- Do not create `.env.example`; this project requires no environment variables or secret configuration.
- Keep the CLI dependency-free unless maintainers explicitly approve a dependency.
- Preserve no-overwrite behavior for generated files.
- Keep network access out of CLI runtime behavior.
- Make scoped changes and avoid unrelated cleanup.
- Update `HANDOFF.md` after meaningful changes.

## Repository Roles

- Maintainer: owns releases, public repository settings, and final merge decisions.
- Manager: turns a request or issue into a scoped task.
- Project lead: makes code or documentation changes and runs verification.
- Reviewer: checks public docs, safety, release readiness, and handoff quality.

These roles are Markdown workflow roles, not an orchestration runtime.

## Verification

Prefer the smallest checks that prove the change:

```bash
python -m unittest discover -s tests
python -m codex_multi_agent_workflow_kit.cli init .tmp_agent_check
python -m codex_multi_agent_workflow_kit.cli check .tmp_agent_check
```

Remove temporary folders after smoke checks.

## Git Hygiene

- Stage only files related to the current task.
- Do not stage `.env`, local caches, temporary smoke folders, or generated bytecode.
- Use conventional commit messages.
- Push only when the maintainer request explicitly asks for it.
